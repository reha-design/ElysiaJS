# 사용자 경험(UX) 설계 원칙 가이드

사용자가 스트레스 없이 직관적으로 서비스를 이용할 수 있도록 만드는 핵심 설계 지침입니다.

---

## 1. 일관성 (Consistency)
사용자는 익숙한 방식에서 안정감을 느낍니다.

- **UI 요소 통일:** 버튼의 모양, 색상, 폰트 스타일 등을 통일합니다.
- **상호작용 패턴:** 같은 기능을 수행하는 동작은 어디서나 동일하게 작동해야 합니다.
- **위치 선정:** 내비게이션 바, 로그아웃 버튼 등 중요한 요소는 예측 가능한 위치에 배치합니다.

## 2. 즉각적인 피드백 (Feedback)
사용자의 모든 행동에는 시스템의 반응이 수반되어야 합니다.

- **상태 알림:** 버튼 클릭 시 로딩 애니메이션, 처리 완료 후 성공/실패 메시지를 즉시 보여줍니다.
- **시각적 변화:** 마우스 호버(Hover) 시 색상 변경 등으로 클릭 가능 여부를 알립니다.
- **반응 속도:** 성능 최적화를 통해 지연 시간을 최소화합니다 (참고: API 설계 원칙의 성능 부분).

## 3. 단순성과 명확성 (Simplicity & Clarity)
불필요한 정보를 제거하고 핵심에 집중하게 합니다.

- **미니멀리즘:** 한 화면에는 하나의 주요 작업(Call to Action)에 집중하게 합니다.
- **쉬운 용어:** 전문 용어나 코드 에러 대신 사용자가 이해할 수 있는 일상 언어를 사용합니다.
- **여백의 활용:** 적절한 여백은 가독성을 높이고 시각적 피로를 줄여줍니다.

## 4. 접근성 (Accessibility)
신체적 제약이나 환경에 상관없이 누구나 이용할 수 있어야 합니다.

- **색상 대비:** 텍스트와 배경의 명도 차이를 충분히 두어 저시력자도 읽을 수 있게 합니다.
- **키보드 접근성:** 마우스 없이 키보드만으로 모든 기능을 이용할 수 있게 설계합니다 (Tab 키 이동 등).
- **스크린 리더 대응:** 이미지에 lt 태그를 넣고, 의미 있는 HTML 태그(Semantic HTML)를 사용합니다.

## 5. 효율성과 유연성 (Efficiency)
숙련된 사용자가 더 빠르게 작업을 마칠 수 있도록 배려합니다.

- **단축키 및 자동 완성:** 검색어나 폼 입력 시 자동 완성 기능을 제공합니다.
- **기본값 설정:** 사용자가 가장 많이 선택하는 항목을 기본값으로 설정하여 입력을 최소화합니다.

## 6. 에러 방지 및 복구 (Error Prevention & Recovery)
에러가 발생하지 않도록 돕고, 발생 시 쉽게 되돌릴 수 있게 합니다.

- **방지책:** 잘못된 형식의 데이터 입력 시 실시간으로 경고를 줍니다 (유효성 검사).
- **명확한 안내:** "에러가 발생했습니다"가 아닌, "비밀번호는 8자 이상이어야 합니다"와 같이 해결책을 제시합니다.
- **실행 취소:** 실수로 데이터를 삭제했을 때 되돌릴 수 있는 기능을 제공합니다.

## 7. 가시적 상태 확인 (Visibility of System Status)
사용자가 현재 자신이 어디에 있고, 무슨 상황인지 알 수 있게 합니다.

- **브레드크럼(Breadcrumbs):** 복잡한 경로에서 현재 위치를 표시합니다.
- **진행 표시바:** 긴 과정(회원가입 등)이 있을 때 현재 몇 번째 단계인지 보여줍니다.
