# 문제 해결 보고서: 데이터베이스 한글 깨짐(???) 현상

## 1. 개요
*   **현상**: `db:studio` 및 API 호출(`curl`) 시, 데이터베이스 내 한글 데이터가 `???` (물음표)로 표시됨.
*   **영향 범위**: 기존에 입력된 모든 한글 게시글 데이터(ID 2~7번).

## 2. 원인 분석
*   **조사 내용**:
    *   `docker-compose.yml`: MySQL 서버 설정에 `utf8mb4` 캐릭터셋이 이미 적용되어 있음을 확인.
    *   `src/db/client.ts`: `mysql2` 연결 설정에 `charset: 'utf8mb4'`가 포함되어 있음을 확인.
    *   `SHOW CREATE TABLE`: 테이블 레벨에서도 `DEFAULT CHARSET=utf8mb4` 설정이 되어 있음을 확인.
*   **최종 결론**:
    인프라 및 코드 설정은 정상이지만, **설정이 완료되기 이전 시점**에 입력된 데이터이거나, **잘못된 인코딩으로 데이터를 보낸 클라이언트 도구**를 통해 삽입된 데이터가 그대로 DB에 저장됨(데이터 자체의 영구적 손상).

## 3. 검증 및 해결 방법
*   **검증 과정**:
    1.  `test_korean.ts` 스크립트를 작성하여 테스트 데이터 삽입.
    2.  정상적으로 한글(`한글 테스트 제목 (최종)`)이 저장되고 조회됨을 확인 (ID 11번 데이터).
*   **해결 방법**:
    1.  앞으로 입력되는 모든 데이터는 현재의 `utf8mb4` 설정에 의해 자동으로 정상 처리됨.
    2.  이미 `???`로 깨진 기존 데이터는 복구가 불가능하므로, 삭제 후 재입력 권장.

## 4. 재발 방지 가이드
1.  새로운 DB 환경 구축 시, 컨테이너 생성과 동시에 `utf8mb4` 설정을 우선적으로 적용할 것.
2.  연결 클라이언트(Drizzle, mysql2 등) 설정에서 `charset` 옵션을 누락하지 말 것.
3.  데이터 입력 도구(IDE의 DB 관리 기능 등) 사용 시 자체 인코딩 설정이 UTF-8인지 확인할 것.
